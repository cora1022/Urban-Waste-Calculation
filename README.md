# 도시 폐기물 분석 대시보드

도시 폐기물 시뮬레이션 CSV를 업로드해서 도시별 폐기물 발생량, 인구 기준 배출량, 시설 유형별 발생량, 재활용 비율, 임시보관 가능일을 비교하는 웹 대시보드이다.

## 자료구조·알고리즘 적용

- 해시맵을 사용해 CSV 데이터를 도시별, 시설 유형별로 빠르게 그룹화했다.
- 도시와 시설 유형을 조합한 키를 만들어 같은 유형의 데이터를 한 번에 누적했다.
- 중복되는 CSV 헤더는 카운팅 방식으로 구분해 같은 이름의 컬럼이 섞이지 않도록 처리했다.
- 선형 누적 합을 사용해 총폐기물, 인구, 폐기물 세부 항목을 한 번의 순회로 계산했다.
- 최소 힙 기반 Top-K 알고리즘을 사용해 폐기물 발생량이 큰 시설 유형과 세부 품목을 추출했다.
- 안정 정렬을 사용해 같은 값이 있을 때도 표의 순서가 자연스럽게 유지되도록 했다.
- 도시 규모 차이를 줄이기 위해 총폐기물을 거주인구와 유효인구 기준으로 정규화했다.
- 가중 점수 알고리즘을 사용해 폐기물량, 재활용 비율, 임시보관 가능일을 종합 점수로 계산했다.
- 임계값 분류를 적용해 효율적, 보통, 비효율적 상태를 자동으로 판정했다.
- Chart.js 시각화에 계산 결과를 연결해 집계 결과가 바로 그래프로 표현되도록 했다.

## 실제 통계와 유사한 점

- 전국 생활폐기물 기준값을 사용해 도시별 1인당 폐기물 발생량을 비교한다.
- 제6차 전국폐기물통계조사의 생활폐기물 기준을 참고 지표로 함께 표시한다.
- 거주인구만 보지 않고 종사인구와 유동인구를 함께 반영해 도시 활동량을 보정한다.
- 주거, 상업, 학교, 산업·공장, 의료, 건설 시설을 나누어 실제 도시 시설 구성을 반영한다.
- 음식물류 폐기물, 재활용가능자원, 종량제봉투 대상 폐기물, 건설폐기물, 의료폐기물, 사업장 일반폐기물처럼 행정 통계에서 쓰이는 분류와 비슷하게 항목을 나누었다.
- 총폐기물만 보여주지 않고 세부 폐기물 구성비를 같이 보여주기 때문에 실제 통계표처럼 원인 분석이 가능하다.
- 재활용가능자원 비율을 따로 계산해 단순 배출량뿐 아니라 자원 회수 가능성도 비교할 수 있다.
- 임시보관 가능일을 계산해 폐기물 발생량과 보관 여력을 함께 볼 수 있다.
- 거주인구 기준과 유효인구 기준을 같이 표시해 상업지나 유동인구가 많은 도시의 특성을 더 현실적으로 해석할 수 있다.
- 시설 유형별 폐기물 발생량을 비교해 어느 도시 구조가 폐기물 증가에 영향을 주는지 확인할 수 있다.

## 장점

- CSV만 업로드하면 별도 서버 없이 바로 분석할 수 있다.
- 도시 A와 도시 B의 차이를 카드, 그래프, 표로 한 번에 확인할 수 있다.
- 자료구조와 알고리즘 개념이 실제 데이터 처리 과정에 연결되어 있다.
- 폐기물 총량, 1인당 배출량, 재활용 비율, 보관 가능일을 함께 보므로 분석 기준이 다양하다.
- 주요 발생원과 세부 품목을 자동으로 찾아주기 때문에 결과 해석이 쉽다.
- 실제 통계 기준을 참고한 시뮬레이션 데이터라 발표나 과제에서 도시 폐기물 분석 예시로 활용하기 좋다.
