# 도시 폐기물 분석 대시보드

도시 폐기물 시뮬레이션 CSV를 업로드해서 도시별 폐기물 발생량, 인구 기준 배출량, 시설 유형별 발생량, 재활용 비율, 임시보관 가능일을 비교하는 웹 대시보드이다.

## 자료구조와 알고리즘

이 대시보드는 업로드된 CSV를 한 번 읽은 뒤, 도시별 통계와 시설 유형별 통계를 빠르게 만들 수 있도록 자료구조와 알고리즘을 분리해서 사용한다. 계산 결과의 의미는 그대로 유지하면서, 데이터 처리 흐름을 컴퓨터공학 수업에서 배우는 개념으로 설명할 수 있게 구성했다.

### 사용한 자료구조

- 배열: CSV 행 목록, 차트 데이터, 표 렌더링 순서를 저장한다.
- 해시 테이블: 도시명, 시설 유형, 폐기물 항목명을 키로 사용해 합계와 개수를 누적한다.
- 복합 키: `도시명 + 시설유형`처럼 여러 속성을 조합한 키로 같은 그룹의 데이터를 한 번에 찾는다.
- 카운터 테이블: 중복 CSV 헤더를 번호로 구분해 같은 이름의 컬럼이 섞이지 않도록 처리한다.
- 최소 힙: 폐기물 발생량이 큰 상위 K개 시설 유형과 세부 품목을 효율적으로 추출한다.
- 정렬 배열: 표와 차트에서 도시, 시설 유형, 폐기물 항목을 안정적인 순서로 보여준다.

### 적용한 알고리즘

- CSV 파싱: 헤더를 읽고 각 행을 객체 형태로 변환한다.
- 그룹화: 해시 테이블을 사용해 도시별, 시설 유형별 데이터를 묶는다.
- 선형 누적 합: 총폐기물, 인구, 재활용가능자원, 세부 폐기물 항목을 한 번의 순회로 합산한다.
- Top-K 추출: 최소 힙을 사용해 전체를 모두 정렬하지 않고 중요한 상위 항목만 찾는다.
- 안정 정렬: 값이 같은 항목이 있을 때도 기존 순서가 자연스럽게 유지되도록 정렬한다.
- 정규화: 도시 규모 차이를 줄이기 위해 총폐기물을 거주인구와 유효인구 기준으로 나눈다.
- 가중 점수 계산: 폐기물량, 재활용 비율, 임시보관 가능일을 종합 점수로 환산한다.
- 임계값 분류: 계산된 지표를 기준으로 효율적, 보통, 비효율적 상태를 자동 판정한다.
- 시각화 매핑: Chart.js가 사용할 수 있는 데이터 배열로 집계 결과를 변환한다.

### 시간 복잡도

`n`을 CSV 행 수, `g`를 그룹 수, `m`을 폐기물 세부 항목 수, `k`를 추출할 상위 항목 수라고 하면 주요 처리 비용은 다음과 같다.

| 기능 | 사용 알고리즘 | 시간 복잡도 |
|---|---|---:|
| CSV 행 변환 | 선형 파싱 | `O(n)` |
| 도시/시설 그룹화 | 해시 테이블 누적 | `O(n)` |
| 폐기물 세부 항목 합산 | 선형 누적 합 | `O(nm)` |
| 상위 K개 항목 추출 | 최소 힙 Top-K | `O(n log k)` |
| 표 순서 정렬 | 안정 정렬 | `O(g log g)` |
| 점수/상태 계산 | 가중 점수 + 임계값 분류 | `O(g)` |
| 차트 데이터 생성 | 배열 매핑 | `O(g + m)` |

## 실제 통계와 유사한 점

- 전국 생활폐기물 기준값을 사용해 도시별 1인당 폐기물 발생량을 비교한다.
- 제6차 전국폐기물통계조사의 생활폐기물 기준을 참고 지표로 함께 표시한다.
- 거주인구만 보지 않고 종사인구와 유동인구를 함께 반영해 도시 활동량을 보정한다.
- 주거, 상업, 학교, 산업·공장, 의료, 건설 시설을 나누어 실제 도시 시설 구성을 반영한다.
- 음식물류 폐기물, 재활용가능자원, 종량제봉투 대상 폐기물, 건설폐기물, 의료폐기물, 사업장 일반폐기물처럼 행정 통계에서 쓰이는 분류와 비슷하게 항목을 나누었다.
- 총폐기물만 보여주지 않고 세부 폐기물 구성비를 같이 보여주기 때문에 실제 통계표처럼 원인 분석이 가능하다.
- 재활용가능자원 비율을 따로 계산해 단순 배출량뿐 아니라 자원 회수 가능성도 비교할 수 있다.
- 임시보관 가능일을 계산해 폐기물 발생량과 보관 여력을 함께 볼 수 있다.
- 거주인구 기준과 유효인구 기준을 같이 표시해 상업지나 유동인구가 많은 도시의 특성을 더 현실적으로 해석할 수 있다.
- 시설 유형별 폐기물 발생량을 비교해 어느 도시 구조가 폐기물 증가에 영향을 주는지 확인할 수 있다.

## 장점

- CSV만 업로드하면 별도 서버 없이 바로 분석할 수 있다.
- 도시 A와 도시 B의 차이를 카드, 그래프, 표로 한 번에 확인할 수 있다.
- 자료구조와 알고리즘 개념이 실제 데이터 처리 과정에 연결되어 있다.
- 폐기물 총량, 1인당 배출량, 재활용 비율, 보관 가능일을 함께 보므로 분석 기준이 다양하다.
- 주요 발생원과 세부 품목을 자동으로 찾아주기 때문에 결과 해석이 쉽다.
- 실제 통계 기준을 참고한 시뮬레이션 데이터라 발표나 과제에서 도시 폐기물 분석 예시로 활용하기 좋다.
